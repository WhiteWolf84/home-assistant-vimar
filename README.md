src=\"https://user-images.githubusercontent.com/51525150/89122129-36b99800-d4c5-11ea-8089-18c2dcab0938.jpg\" width=\"300\">\n\n## 🌟 New Feature: Time-Based Covers (Shutters) Tracking\n\nThis branch (`timed-shutters-legacy-mode`) introduces an advanced position tracking engine for Vimar covers/shutters that lack native positional feedback.\n\n**Key features of the new engine:**\n- **Position Estimation:** Accurately estimates cover position (0-100%) based on configured up/down travel times.\n- **Relay Delay Compensation:** Automatically compensates for the Vimar web server HTTP/Relay delay (0.5s) to provide better tracking accuracy.\n- **Database Optimization:** Reduces HA database spam by throttling UI updates, avoiding interface flickering and saving I/O operations.\n- **Auto-calibration:** Re-synchronizes position with physical end-stops when a full motion (0% or 100%) is executed via UI or physical buttons.\n- **Four Operating Modes:**\n  - `legacy`: (Default) Original behavior. Does not track time, relies only on native Vimar sensors (if present).\n  - `native`: Strictly uses Vimar positional sensors. Falls back to \"assumed state\" if unavailable.\n  - `time_based`: Forces the time-based calculation engine, ignoring Vimar positional sensors.\n  - `auto`: Uses native sensors if available; automatically falls back to time-based tracking if no hardware sensor is detected.\n\n## Hardware Requirements\n\n* **[Vimar 01945 - Web server By-me](https://www.vimar.com/de/int/catalog/obsolete/index/code/R01945)** or\n* **[Vimar 01946 - Web server Light By-me](https://www.vimar.com/en/int/catalog/product/index/code/R01946)**\n\n> **Note:** Tested with firmware versions v2.5 to v2.8. Ensure you have a full backup of your Vimar database before performing web server firmware upgrades.\n\n## Installation\n\n### HACS (Recommended)\n1. Open HACS in Home Assistant.\n2. Add this repository as a Custom Repository (Integration category).\n3. Install the integration and restart Home Assistant.\n\n### Manual Installation\nDownload the latest release and copy the `custom_components/vimar` folder into your Home Assistant `custom_components` directory.\n\n## Configuration\n\nConfiguration is fully managed via the Home Assistant UI.\n1. Go to **Settings > Devices & Services**.\n2. Click **Add Integration** and search for **Vimar By-Me Hub**.\n3. Enter your web server credentials (IP, username, password).\n\n### Cover Travel Times Setup\nOnce added, you can configure individual travel times for your covers to ensure accurate tracking:\n1. Go to **Developer Tools > Services**.\n2. Select the `vimar.set_travel_times` service.\n3. Target the specific cover entity.\n4. Input the precise `travel_time_up` and `travel_time_down` (in seconds).\n\n## Limitations & Disclaimer\n\nThis integration supports reading and controlling lights, dimmers, audio devices, energy meters, covers, fans, switches, climates, and scenes. Other specific hardware types may not be fully supported.\n\n**DISCLAIMER: THIS IS A COMMUNITY-DRIVEN PROJECT.** Use at your own risk. It mimics HTTP calls usually made through the official Vimar By-me web interface.\n\n## Troubleshooting\n\nIf you experience issues, enable debug logging by adding the following to your `configuration.yaml`:\n\n```yaml\nlogger:\n  default: warning\n  logs:\n    custom_components.vimar: debug\n```\n\nIf the Vimar Web server GUI becomes unresponsive while Home Assistant is connected, please create a **dedicated user** on your Vimar web server specifically for this integration, as the server tends to drop sessions if the same user logs in from multiple clients.\n\n## Contributing\n\nContributions, bug reports, and pull requests are always welcome.
